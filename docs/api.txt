{

    public string? PhoneNumber { get; set; }
    public string? Email { get; set; } // todo: persist

    public AddressDto? Address { get; set; }
}

public class AddressDto
{
    public string? ExternalId { get; set; }
    public string? AddressName { get; set; }
    public string? AddressData { get; set; }
}

public class LessonSuggestionDto
{
    public string? ExternalId { get; set; }

    public DateTimeOffset? SuggestedEnd { get; set; }

Pobieranie uczniów:
GET /api/student - dla pobrania wszystkich uczniów

GET /api/student?includeDeleted=True  - z uwzględnieniem usuniętych (jeszcze nie ma na devie)
Zwraca: Listę StudentDto

Tworzenie ucznia:
POST /api/student
Możliwe schematy Request Body :
{
    "Name": "Basia",
    "Surname": "Be",
    "PhoneNumber": "123456",
    "Email": "jakiś@mail.cośtam",
    "Address": {
        "ExternalId": "2"
    }
}
Powyższe utworzy ucznia przypisując do niego istniejący już adres o ID 2
{
    "Name": "x",
    "Surname": "y",
    "PhoneNumber": "123456",
    "Email": "jakiś@mail.cośtam",
    "Address": {
        "AddressName": "nowy",
        "AddressData": "ulica"
    }
}
Powyższe utworzy nowego ucznia oraz utworzy nowy adres
Zwraca: kod sukcesu/błędu

Update ucznia:
PATCH /api/student/{studentExternalId}
Możliwe schematy request body:
{
    "Name": "x",
    "Surname": "y",
    "PhoneNumber": "123456",
    "Email": "jakiś@mail.cośtam",
    "Address": {
        "AddressName": "nowy",
        "AddressData": "ulica"
    }
}
Body może zawierać dowolne informacje spośród powyższych, pominięte dane nie zostaną zmienione.
UWAGA: Przy podaniu adresu jak poniżej:
{
    "Address": {
        "ExternalId": "2",
        "AddressName": "ZmianaNazwyIstniejącegoAdresu"
    }
}
Zostanie zmodyfikowany adres o ID 2
Natomiast przy podaniu adresu w poniższej formie:
{
    "Address": {
        "AddressName": "NazwaNowegoAdresu",
        "AddressData": "DaneNowegoAdresu"
    }
}
Zostanie utworzony nowy adres.
W sytuacji poniższej:
{
    "Address": {
        "AddressName": "NazwaNowegoAdresu"
    }
}
Zostanie utworzony nowy adres, a "AddressData" zostanie skopiowane z dotychczasowego adresu ucznia
Zwraca: kod sukcesu/błędu

Usuwanie ucznia:

Zwraca: kod sukcesu/błędu
